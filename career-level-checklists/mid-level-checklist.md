# Mid-Level Engineer Checklist (2025)

You now own features end-to-end and stop things from breaking in production.

| Category                     | Must-Master Frameworks / Skills                                           | ✓ |
|------------------------------|---------------------------------------------------------------------------|---|
| Architecture                 | Creates C4 Level 1–3 diagrams independently                               |   |
|                              | Applies Hexagonal/Clean Architecture in real projects                     |   |
|                              | Uses DDD building blocks (Entities, Value Objects, Aggregates)            |   |
| Reliability                  | Understands and uses SRE concepts: SLO/SLI/Error Budget                   |   |
|                              | Sets up alerting + on-call runbooks                                       |   |
| Testing                      | Writes integration + contract tests                                       |   |
|                              | Uses property-based testing when useful                                   |   |
| Distributed Systems          | Knows at-least-once vs at-most-once vs exactly-once                       |   |
|                              | Implements retry + circuit breaker patterns                               |   |
| Event-Driven                 | Designs with Kafka/RabbitMQ + Saga pattern                                |   |
| APIs & Contracts             | Uses OpenAPI + asyncAPI, validates contracts                              |   |
| CI/CD & DevOps               | Owns full pipelines (build → test → deploy → rollback)                    |   |
|                              | Writes IaC (Terraform, Pulumi, CDK)                                       |   |
| Data Modeling                | Chooses between SQL vs NoSQL vs Event Store with reasoning               |   |
|                              | Understands CQRS and when to apply it                                     |   |
| Documentation                | Writes and maintains ADRs (Architecture Decision Records)                 |   |
| Mentoring                    | Reviews PRs and onboard new juniors                                       |   |

~ 2–5 years. When 85 %+ checked → you’re senior-ready.
